# 九路围棋规则

本文档为 `BoardArena/go_9x9` 的规则介绍。环境实现标准 9x9 围棋的核心规则，使用中国式面积计分。

## 基本设置

- 支持玩家数：固定 `2` 人。
- 玩家 `0`：黑方，先手。
- 玩家 `1`：白方，后手。
- 棋盘：9x9。
- 初始局面：空棋盘。
- 默认贴目：白方 `6.5` 目。
- 棋盘坐标：文件为 `a..i`，横线为 `1..9`。

## 行动格式

所有动作使用字符串：

- 普通落子：棋盘坐标，例如 `e5`、`c7`。
- 跳过：`PASS`。

`PASS` 始终合法。双方连续 `PASS` 后终局并计分。

## 合法落子

一次落子必须满足：

- 坐标在棋盘内。
- 目标点为空。
- 落子后会移除所有无气的相邻对方棋串。
- 禁止自杀：如果落子并提子后自己的新棋串无气，则该落子非法。
- 简单 ko：如果落子后的整盘局面复现上一手之前的整盘局面，则该落子非法。

当前实现不包含复杂长生/超劫规则，只处理直接 ko。

## 终局和计分

满足任一条件时结束：

- 双方连续 `PASS`。
- 棋盘已满。
- Bot 对战中达到环境 `max_plies` 上限时，以 `turn_limit` 截断。

正常终局使用中国式面积计分：

```text
黑方得分 = 黑子数 + 黑方围空
白方得分 = 白子数 + 白方围空 + 贴目
```

空点区域如果只接触一方棋子，算作该方围空；如果同时接触双方棋子或不接触任何棋子，算作中立点。

## 异常结束

Bot 对战中：

- Bot 抛异常：状态为 `bot_exception`，该 Bot 判负。
- Bot 返回非法动作：状态为 `invalid_action`，该 Bot 判负。
- Bot 单步决策超时：状态为 `timeout`，该 Bot 判负。

常见 `status`：

- `two_passes`
- `board_full`
- `turn_limit`
- `invalid_action`
- `bot_exception`
- `timeout`
